# UI Agent Learning Loop — BracketFlow

## Lección principal

Los agentes deben distinguir entre:
- cambiar layout,
- cambiar copy,
- cambiar fondo,
- cambiar atmósfera,
- cambiar producto.

En BracketFlow, una mejora visual no puede romper la lectura operativa.

## Regla owner

No hay commit visual sin aprobación explícita de Nelson en screenshots reales.

## Patrón fallido

El agente ve un área vacía y agrega decoración local.

## Patrón correcto

El agente debe extender el sistema visual existente sin competir con la acción principal.
